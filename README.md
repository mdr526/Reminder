# یادآور (Reminder PWA)

اپلیکیشن وب یادآوری کارها با تقویم جلالی (شمسی)، تکرار هفتگی، و هشدارهای مرحله‌ای — به‌صورت PWA قابل‌نصب روی موبایل.

## ساختار فایل‌ها

```
pwa/
├── index.html      همه‌چیز اینجاست: HTML + CSS + JS در یک فایل واحد
├── manifest.json   تنظیمات نصب PWA (نام، آیکون‌ها، تم رنگی)
├── sw.js           Service Worker برای کش آفلاین (استراتژی network-first)
└── icons/          آیکون‌های اپ در سایزهای مختلف
```

`index.html` عمداً تک‌فایلی نگه داشته شده (بدون فایل جدا برای CSS/JS) تا هم توسعه و هم توزیعش ساده باشه.

## ویژگی‌ها

- ثبت یادآور با عنوان، تاریخ (تقویم جلالی)، و ساعت
- تکرار هفتگی روی روزهای دلخواه هفته
- سه مرحله هشدار برای هر کار: ۱ ساعت مانده، ۵ دقیقه مانده، سررسید — هرکدام فقط وقتی نمایش داده می‌شن که واقعاً از ابتدا برای اون کار وقت کافی وجود داشته (یعنی کاری که با ۴۰ دقیقه باقی‌مانده ثبت بشه، هشدار «۱ ساعت مانده» رو نمی‌گیره)
- نمایش شمارش معکوس زنده (روز/ساعت/دقیقه/ثانیه) که هر ثانیه به‌روز می‌شه
- آرشیو خودکار کارهای گذشته + امکان مشاهده و پاک‌سازی
- حالت روشن/تاریک
- نصب به‌صورت PWA (Add to Home Screen) با پشتیبانی آفلاین

## نصب و اجرا

1. کل پوشه‌ی `pwa/` رو روی یک هاست استاتیک (GitHub Pages، Netlify، Vercel، یا هر وب‌سروری) آپلود کنید. باز کردن مستقیم فایل با `file://` هم برای تست کار می‌کنه، ولی Service Worker و نصب PWA فقط روی `https://` (یا `localhost`) فعال می‌شن.
2. در مرورگر موبایل باز کنید → گزینه‌ی «افزودن به صفحه اصلی» / «Add to Home Screen» رو بزنید.
3. موقع اولین استفاده، اجازه‌ی نوتیفیکیشن رو از طریق بنر بالای صفحه فعال کنید.

## محدودیت‌های شناخته‌شده

- **iOS:** نوتیفیکیشن محلی فقط وقتی اپ در پیش‌زمینه یا به‌تازگی در پس‌زمینه است کار می‌کنه. iOS اجرای جاوااسکریپت رو در پس‌زمینه به‌شدت محدود می‌کنه و این اپ از Web Push واقعی (که نیاز به سرور دارد) استفاده نمی‌کنه؛ پس اگر اپ برای مدت طولانی بسته/کشته بشه، هشدارها تا باز شدن مجدد اپ نمایش داده نمی‌شن.
- **اندروید:** تایمرها ممکنه در پس‌زمینه throttle بشن؛ منطق هشدار طوری طراحی شده که با تأخیر هم هشدار رو از دست نده (نه بازه‌ی باریک، بلکه بازه‌ی پهن + پرچم «قبلاً اعلام‌شده» در localStorage).
- **مدول انتخاب ساعت:** در نمایشگرهای خیلی کوتاه (مثلاً حالت افقی گوشی، ارتفاع نمایشگر کمتر از ~۴۲۰ پیکسل)، ممکنه دکمه‌های «تأیید/انصراف» این مدول از پایین بریده بشن. (برطرف‌نشده؛ برای جزئیات به تاریخچه‌ی بررسی کد مراجعه کنید.)
- **کنتراست متن:** متن دو دیالوگ تأیید حذف («حذف یادآور» و «حذف همه») در تم روشن به‌خاطر رنگ ثابت، کنتراست پایینی داره. (برطرف‌نشده.)

## ذخیره‌سازی داده‌ها

همه‌چیز فقط در `localStorage` مرورگر/دستگاه ذخیره می‌شه (کلیدهای `reminders_v2`، `past_reminders_v2`، `notified_map_v1`، `reminder-theme`). هیچ سروری در کار نیست؛ پاک کردن داده‌های مرورگر یا نصب مجدد اپ باعث از دست رفتن یادآورها می‌شه.

## توسعه

برای تغییر ظاهر یا منطق، مستقیماً `index.html` رو ویرایش کنید — بخش `<style>` در ابتدای فایل و بخش `<script>` در انتهای فایل قرار داره. کلیدی‌ترین توابع:

| تابع | کاربرد |
|---|---|
| `checkReminders()` | هر ثانیه اجرا می‌شه، هشدارهای سه‌مرحله‌ای رو چک می‌کنه |
| `getCountdownText()` | فرمت متن شمارش معکوس (روز/ساعت/دقیقه/ثانیه) |
| `moveDueToArchive()` | آرشیو خودکار کارهای سررسیده و پیشروی تکرارها |
| `renderList()` / `updateCountdownBadges()` | رندر کامل لیست / آپدیت سبک فقط بج‌های زمان |
